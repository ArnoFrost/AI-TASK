## 自动行为（无需用户指令）
1. **阅读任务列表** - 了解当前进度，避免重复工作
2. **自动创建任务** - 用户描述需求时，自动生成任务文档到 tasks/
3. **自动命名编号** - 格式: {DATE}-{SEQ}_[标签]名称.md，用户无需关心
4. **自动更新状态** - 任务完成后更新本文件任务列表

## 任务文档自动生成规则
- 文件名: YYYYMMDD-NNN_[标签]任务名.md (AI自动生成)
- 标签: 根据任务内容自动判断（详见 ../../SPEC.md#标签类型）
- 编号: **全局递增，不按日期重置**，取 tasks/ 目录最大 NNN + 1

## 文件夹任务规则（预期产出 ≥2 文档或多阶段/多角色时使用）
- 入口文件: **必须**命名为 `task_plan.md`，不得使用其他名称
- 子文件格式: `task_{N}_{主题}.md`，编号从 1 递增，平铺在文件夹根部
- 常见错误: ❌ `评审主文档.md` ❌ `AAR规格.md` → ✅ `task_plan.md` + `task_1_xxx.md`

## 用户只需要
- 描述要做什么（自然语言）
- AI 自动完成：创建任务 → 执行 → 更新状态 → 提交

## 🧭 任务命名与结构

**统一编号格式**：`YYYYMMDD-NNN_[标签]任务名称`
- `NNN` 为**全局递增**，不随日期重置
- **标签**：`[功能/优化/修复/排查/文档/调研/评审/专项]`

### 单文件任务（简单任务）

    tasks/
    └── 20260211-003_[排查]某某问题分析.md

### 文件夹任务（复杂/专项任务）

预期产出 >= 2 个文档，或涉及多阶段/多角色时使用。

    tasks/
    └── 20260226-001_[专项]某某专项/
        ├── task_plan.md          # 总体规划（必须）
        ├── task_1_xxx.md
        └── ...

| 规则 | 说明 |
|------|------|
| 入口文件 | `task_plan.md` 必须存在 |
| 子文件命名 | `task_{N}_{主题}.md` |
| index.md 引用 | 链接到文件夹目录即可 |
