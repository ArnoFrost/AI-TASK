执行 AI-TASK 任务管理操作。

## 用户输入

$ARGUMENTS

## 操作类型识别

根据用户输入判断操作类型：
- `create [标签] 任务名` 或 `新建 [标签] 任务名` → 创建任务
- `list` 或 `列表` → 列出任务
- `update 任务编号 内容` → 更新任务
- `done 任务编号` 或 `完成 任务编号` → 标记完成
- 无参数 → 显示帮助

## 执行规范

### 路径约定

- 任务目录: `ai-task/tasks/`
- 项目信息: `ai-task/project.yaml`
- 项目入口: `ai-task/index.md`

### 文件命名

```
{YYYYMMDD}-{NNN}_[标签]任务名称.md
```

- 日期: 当天日期 YYYYMMDD
- 序号: 三位数，**日内递增**，每天从 001 开始，取当天最大 NNN + 1
- 标签: `[功能]` `[优化]` `[修复]` `[排查]` `[文档]` `[调研]` `[技术方案]` `[规范]` `[下线]` `[清理]` `[梳理]` `[测试]` `[评审]` `[架构]` `[集成]` `[同步]`（详见 [SPEC.md](../../SPEC.md#标签类型)）

### 创建任务流程

1. 读取 `ai-task/project.yaml` 获取项目信息
2. 扫描 `ai-task/tasks/` 确定全局序号
3. 按模板创建任务文件
4. 更新 `ai-task/index.md` 任务列表

### 任务模板

详见 [task-template.md](../../task-template.md)（唯一权威模板）

### 列出任务流程

1. 扫描 `ai-task/tasks/` 目录
2. 按状态分组展示（进行中 / 已完成）
3. 显示任务编号、标签、名称

### 更新任务流程

1. 定位任务文件
2. 更新进度记录表格
3. 如需要，更新状态

### 标记完成流程

1. 定位任务文件
2. 更新状态为 `✅ 已完成`
3. 更新 `ai-task/index.md`

## 帮助信息

```
/task 命令用法:

  /task create [标签] 任务名称    创建新任务
  /task list                     列出所有任务
  /task update 编号 进度说明      更新任务进度
  /task done 编号                标记任务完成

标签类型:
  [功能] [优化] [修复] [排查] [文档] [调研] [技术方案] [规范]
  [下线] [清理] [梳理] [测试] [评审] [架构] [集成] [同步]

示例:
  /task create [功能] 用户登录模块
  /task list
  /task update 001 完成接口对接
  /task done 001
```
