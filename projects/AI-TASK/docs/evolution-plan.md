# AI-TASK 演进计划（索引与归档规范）

## 🎯 目标

- 明确 `README.md` / `SPEC.md` 为规范入口，`index.md` 仅负责索引与状态记录
- 建立可复用的索引结构与归档规则，推动跨项目一致性
- 在 AI-TASK 内部先行落地，再推广至其他项目

---

## 🧩 现状与问题

- 不同项目把 `index.md` 当作规范文档，导致索引过长、信息冗余
- 归档规则缺失或未执行，任务目录与索引长期不一致
- 项目初始化模板未强制区分 README 与 index 职责

---

## ✅ 目标输出

1. **统一结构**：README/SPEC 归规范，index 归索引
2. **索引模板**：固定“进行中/最近完成/历史归档”结构
3. **归档规则**：`tasks/` 保留近 3 天 + 进行中，超期自动移至 `archive/YYYY-MM/`
4. **推广机制**：在 `init-project.sh` 和模板中内置规范片段

---

## 🛠️ 演进路径

| 阶段 | 范围 | 关键动作 | 交付物 |
|------|------|----------|--------|
| P0 | AI-TASK | index 增加职责说明 + 演进计划文档 | `index.md` + `docs/evolution-plan.md` |
| P1 | 模板层 | 项目模板加入索引模板与归档规则 | `templates/` + `init-project.sh` |
| P2 | 存量项目 | 执行迁移与归档清理 | 各项目 `index.md`/`archive/` |

## 📊 评估指标

- 80%+ 项目完成 README/index 职责分离
- 归档规则在新增任务时自动执行
- 索引长度控制在可扫描范围（建议 ≤ 80 行）
