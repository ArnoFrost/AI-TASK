## 🔧 演进方案

### 三层架构设计

```
┌─────────────────────────────────────────┐
│  IDE 适配层 (templates/{ide}/commands/)  │  ← 薄层：仅适配语法
├─────────────────────────────────────────┤
│  核心能力层 (skills/)                    │  ← 厚层：实际逻辑
├─────────────────────────────────────────┤
│  模板层 (templates/*.md)                 │  ← 静态资源
└─────────────────────────────────────────┘
```

### Phase 1: 统一 Skills 核心层

**目标**：skills/ 成为唯一的能力定义层

```
skills/
├── README.md                 # 技能索引
├── task-management/
│   ├── SKILL.md              # 能力定义（agentskills.io 格式）
│   └── references/           # 参考文档（按需加载）
├── project-init/
│   └── SKILL.md
└── ...
```

**SKILL.md 格式**（对齐 agentskills.io）：

```yaml
---
name: task-management
description: |
  任务创建、更新、查询。
  Use when: 创建任务、任务管理、task create
---

# 任务管理技能

## 🎯 能力范围
...

## 🔧 操作指令
...
```

### Phase 2: Commands 适配层瘦身

**目标**：commands/ 只做语法适配，引用 skills/

```
templates/
├── claude/
│   ├── CLAUDE.md             # IDE 入口
│   └── commands/
│       └── task.md           # 引用 skills/task-management
├── codebuddy/
│   ├── CODEBUDDY.md
│   └── commands/
│       └── task.md           # 引用 skills/task-management（同内容）
└── shared-commands/          # 🆕 共享命令层
    ├── task.md
    ├── init_sub_project.md
    └── archive.md
```

**命令模板简化**：

```markdown
执行任务管理操作。

## 用户输入
$ARGUMENTS

## 执行规范
详见：[skills/task-management/SKILL.md](../../skills/task-management/SKILL.md)
```

### Phase 3: 目录清理

- [ ] 删除根目录异常的 `TVKMM/` 目录
- [ ] 统一 templates/shared-commands/ 为公共命令
- [ ] claude/ 和 codebuddy/ 仅保留入口文件
