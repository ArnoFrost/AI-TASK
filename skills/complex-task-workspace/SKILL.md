# 复杂任务工作区 (Complex Task Workspace)

> 当任务复杂度超出单文件承载能力时，提供标准化的目录结构与导航机制

## 触发条件

满足以下**任一**条件时，应使用文件夹任务而非单文件任务：

| 条件 | 示例 |
|------|------|
| 预期产出 >= 2 个文档 | 方案 + 评审报告 |
| 多阶段交付 | 调研 -> 设计 -> 实现 -> 验证 |
| 多角色协作 | 评审、梳理等需要多视角的任务 |
| 用户明确要求 | "创建一个任务工作区" |

**不满足以上条件的简单任务**：继续使用单文件 `.md`，不受本技能影响。

## 目录约定

提供 **minimal** 和 **full** 两个变体，根据实际需求选择。

### Minimal 变体（推荐起步）

```
tasks/{编号}_[标签]任务名/
├── task_plan.md      # 任务入口（目标/范围/状态/产物索引）—— 必须，统一命名
├── task_1_xxx.md     # 子文件，平铺在根部
└── task_2_xxx.md
```

适用场景：大部分多文档任务，先用 minimal 开始，按需扩展。

### Full 变体

```
tasks/{编号}_[标签]任务名/
├── task_plan.md      # 任务入口 —— 必须，统一命名
├── task_1_xxx.md     # 子文件，平铺在根部
├── task_2_xxx.md
├── artifacts/        # 阶段产物（方案、评审报告、决策记录）—— 按需
│   ├── proposal.md
│   └── final-delivery.md
├── notes/            # 过程记录（讨论、假设、参考）—— 按需
└── templog/          # 原始日志/临时数据 —— 按需
```

适用场景：大型任务、需要产物与过程分离、跨会话协作。

### 选择决策

```
预期产出 >= 2 文档？
├── 否 → 单文件任务
└── 是 → 需要产物/过程分离？
    ├── 否 → Minimal 变体
    └── 是 → Full 变体
```

## 入口索引模板 (task_plan.md)

文件夹任务**必须**有 `task_plan.md` 作为入口，集中描述任务全貌：

```markdown
# {任务名称}

> {一句话目标}

## Problem / Objective
{问题定义与目标}

## Scope
- In scope: ...
- Out of scope: ...

## Inputs
- [ ] {输入材料/链接}

## Milestones
| 阶段 | 目标 | 状态 | 产物 |
|------|------|------|------|
| M1 | ... | 🔄 | `artifacts/xxx.md` 或 `tasks/task_1_xxx.md` |

## Artifacts Map
| 产物 | 路径 | 状态 |
|------|------|------|
| ... | `artifacts/xxx.md` | 📝 草稿 |

## Risks & Open Questions
- {风险/未决}

## Decision Log
| 日期 | 决策 | 理由 |
|------|------|------|

## Changelog
| 日期 | 变更 |
|------|------|
```

## 操作指令

### 创建文件夹任务

当触发条件满足时：

1. 在 `tasks/` 下创建以 `{编号}_[标签]任务名/` 命名的目录
2. 创建 `task_plan.md`（使用上述模板）—— 入口文件，统一命名，不可更改
3. 根据选择的变体按需创建 `artifacts/` / `notes/` 等子目录
4. 在项目 `index.md` 任务列表中添加链接（指向文件夹）

### 子任务命名

```
tasks/task_{N}_{主题}.md
```

- `N` 从 1 开始递增
- 主题用简短英文或中文描述

### 归档

文件夹任务**整体**移入 `archive/YYYY-MM/`：

```bash
mv tasks/{编号}_[标签]任务名/ archive/YYYY-MM/
```

## 向后兼容

| 场景 | 处理方式 |
|------|---------|
| 简单任务 | 保持单文件 `.md` 不变 |
| 已有文件夹任务（旧格式） | 可选迁移，旧格式继续工作 |
| 新任务不确定复杂度 | 先用单文件，膨胀后再升级为文件夹 |
