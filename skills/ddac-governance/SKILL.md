# DDAC 自治理协议

> Document-Driven AI Collaboration - 让 AI 学会自己理解规范

## 核心理念

| 能力 | 说明 |
|------|------|
| **自我理解** | 读取规范文档，理解工作标准 |
| **自主执行** | 根据规范自动推导步骤 |
| **持续协作** | 跨会话保持上下文 |
| **自我优化** | 反馈闭环完善规范 |

## 进入项目上下文时

当用户通过 `@projects/{PROJECT}/index.md` 引入项目时：

1. **阅读任务列表** - 了解当前进度和活跃任务
2. **识别沉淀触发** - 判断是否需要创建任务文档
3. **执行并记录** - 任务完成后更新状态
4. **反馈优化** - 发现规范缺失时提议补充

## 任务沉淀触发条件

**满足以下任一条件时，必须创建任务文档：**

- 用户提出多步骤优化/改进计划
- 讨论内容涉及 3+ 个文件修改
- 用户说"梳理"、"规划"、"方案"、"改进"
- 用户要求"沉淀"、"记录"、"保存"

## 自治理原则

```
projects/{PROJECT}/  = 项目自治空间
项目代号 = 目录名   = 管理自身任务
```

**示例**：`projects/AI-TASK/` 管理 AI-TASK 项目自身的任务

## 路书四大法则

| 法则 | 执行要点 |
|------|---------|
| **倒排** | 最新任务在前 |
| **归档** | 完成即归档，活跃 ≤ 3 |
| **更新** | 实时更新进度 |
| **格式** | `YYYYMMDD-NNN_[TAG]NAME.md` |

## 任务文档结构

```markdown
# [标签] 任务名称

**任务编号**: YYYYMMDD-NNN
**状态**: 🔄 进行中 / ✅ 已完成
**优先级**: 🔴 高 / 🟡 中 / 🟢 低

## 📋 任务目标
- [ ] 目标 1
- [ ] 目标 2

## 📊 进度记录
| 时间 | 进度 | 说明 |
```

## 协作闭环

```
用户引入上下文 → AI 阅读规范 → 识别任务 → 创建文档 → 执行任务 → 更新状态 → 反馈优化
```
