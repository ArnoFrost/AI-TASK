# 任务管理技能

## 🎯 能力范围

- 创建新任务文档
- 更新任务进度和状态
- 查询任务列表
- 标记任务完成
- 任务归档

## 📐 命名规范

```
{日期}-{序号}_[标签]任务名称.md
```

- **日期**: `YYYYMMDD`
- **序号**: 三位数，每日每项目从 `001` 开始
- **标签**: `[功能]` `[优化]` `[修复]` `[排查]` `[文档]` `[调研]` `[技术方案]` `[规范]` `[下线]` `[测试]` `[依赖]` `[任务]`

## 📂 路径约定

```
AI-TASK/projects/{PROJECT_CODE}/tasks/{TASK_FILE}.md
```

| 项目代号 | 项目名称 | 路径 |
|----------|----------|------|
| EXAMPLE | 示例项目 | `projects/EXAMPLE/tasks/` |
| _shared | 通用任务 | `projects/_shared/tasks/` |

## 🔧 操作指令

### 创建任务

```markdown
## 输入
项目: {PROJECT_CODE}
标签: [功能/优化/修复/...]
名称: {TASK_NAME}
目标:
1. {GOAL_1}
2. {GOAL_2}

## 执行步骤
1. 检查当日该项目已有任务数量，确定序号
2. 使用任务模板创建文件
3. 填充任务信息
4. 更新项目 index.md 的任务列表
```

### 更新任务

```markdown
## 输入
任务文件: {TASK_FILE}
进度: {PROGRESS}%
完成内容: {DONE}
下一步: {NEXT}

## 执行步骤
1. 读取任务文件
2. 更新进度记录表格
3. 更新任务状态（如需要）
```

### 标记完成

```markdown
## 输入
任务文件: {TASK_FILE}
验证结果:
- {CHECK_1}: ✅/❌
- {CHECK_2}: ✅/❌

## 执行步骤
1. 更新状态为 ✅ 已完成
2. 填写验证结果
3. 更新总结与收获
4. 更新项目 index.md
```
