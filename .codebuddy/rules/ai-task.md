# AI-TASK 协作规范

## 核心原则

1. **任务驱动**: 所有工作以任务为单位进行跟踪
2. **文档即代码**: 任务文档是工作的核心产出
3. **软链接架构**: 通过 `ai-task/` 软链接连接项目与 AI-TASK 仓库

## 目录结构约定

```
项目根目录/
├── ai-task/              # 软链接 → AI-TASK/projects/{CODE}
│   ├── project.yaml      # 项目元数据
│   ├── index.md          # 项目入口
│   ├── tasks/            # 任务目录
│   ├── docs/             # 文档目录
│   └── archive/          # 归档目录
├── CODEBUDDY.md          # AI 协作入口
└── .codebuddy/
    ├── commands/         # 自定义命令
    └── rules/            # 项目规则
```

## 任务文件规范

### 命名格式

```
{YYYYMMDD}-{NNN}_[标签]任务名称.md
```

### 标签类型

| 标签 | 用途 |
|------|------|
| `[功能]` | 新功能开发 |
| `[优化]` | 性能/代码优化 |
| `[修复]` | Bug 修复 |
| `[排查]` | 问题排查分析 |
| `[文档]` | 文档编写 |
| `[调研]` | 技术调研 |
| `[技术方案]` | 方案设计 |
| `[规范]` | 规范制定 |

### 状态标识

- 🔄 进行中
- ✅ 已完成
- ⏸️ 暂停
- ❌ 取消

## 工作流程

1. **开始工作**: `/task create [标签] 任务名`
2. **更新进度**: `/task update 编号 说明`
3. **完成任务**: `/task done 编号`
4. **归档任务**: `/archive 编号`

## 注意事项

- 每个任务文件应包含完整的上下文信息
- 进度记录表格实时更新
- 涉及文件表格记录所有修改
- 任务完成后填写总结与收获
