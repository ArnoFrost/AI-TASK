# 项目规则库

> 项目级别的代码规范、架构约定和最佳实践

## 🎯 设计理念

Rules 是项目特定的约束和指南，帮助 AI 在特定项目上下文中做出更准确的决策。

### 与 Skills 的区别

| 维度 | Skills | Rules |
|------|--------|-------|
| 作用 | 定义"如何做" | 定义"做什么/不做什么" |
| 范围 | 通用能力 | 项目特定 |
| 加载 | 按需激活 | 项目上下文自动加载 |
| 示例 | 任务管理、日报生成 | 代码规范、架构约定 |

## 🔧 使用方式

### 自动加载

当 AI 检测到当前工作在某个项目时，应自动加载对应规则：

```
用户: 帮我优化某个功能
AI: [检测到项目，加载项目规则]
    根据项目规范，优化应注意：
    1. ...
    2. ...
```

### 手动查看

```
用户: 查看项目规则
AI: [显示 rules/{PROJECT_CODE}.md 内容]
```

## 📐 规则文件格式

```markdown
---
name: project-rules
description: 规则描述
project: PROJECT_CODE
---

# 项目名称 项目规则

## 🏗️ 架构约定
...

## 📐 代码规范
...

## 🔥 性能关注点
...

## 📋 任务标签映射
...

## 🔗 参考文档
...
```

## 🆕 创建新规则

1. 在 `rules/` 下创建 `{PROJECT_CODE}.md`
2. 填写 YAML 前置元数据
3. 编写架构约定、代码规范等
4. 更新本文件的规则列表
